# Experiment Structure

This section describes a simple and recommended structure for organizing an experiment that will be executed on the server.

The idea is to clearly separate three different elements:

- **the execution environment**
- **the experiment code**
- **the data and results**

This separation makes it easier to:

- reproduce experiments
- change configurations easily
- avoid unnecessary rebuilding of Docker images

## Dockerfile

The **Dockerfile** defines the environment in which the experiment will run.

This includes:

- the base image
- the dependencies
- the required files
- the default command

## docker-compose

The `docker-compose` file defines how the experiment is executed.

It specifies aspects such as:

- how the image should be built
- which environment variables to use
- which directories should be mounted as volumes
- whether the experiment uses a GPU

## Running tests

Sometimes it is useful to run a variant of the experiment, for example to:

- test changes quickly
- run tests
- validate that the environment works correctly

For this purpose we can create a second `docker-compose` file.

This file can modify aspects such as:

- the script that is executed. *IMPORTANT:* Notice how the `command` field is used to change the script that runs when the experiment starts
- the experiment parameters
- specific configurations for testing
